# 🔄 CI/CD – Integração Contínua e Entrega Contínua

Resumo simples e direto dos conceitos de CI/CD aplicados ao trabalho de um QA Automation Engineer.

---

## 🧩 O que é CI (Continuous Integration)
- Processo onde o código é integrado no repositório várias vezes ao dia.  
- Cada integração dispara **build + testes automáticos**.  
- Objetivo: detetar erros cedo e garantir estabilidade.  
- QA participa garantindo que testes UI/API estão integrados no pipeline.  
- Ferramentas comuns: Jenkins, GitHub Actions, Azure Pipelines.

---

## 🧩 O que é CD (Continuous Delivery/Deployment)
- Automatiza a entrega do software para ambientes de teste ou produção.  
- Garante que cada versão estável pode ser lançada rapidamente.  
- QA valida qualidade antes da promoção entre ambientes.  
- Pipelines podem incluir testes funcionais, integração e regressão.

---

## 🧩 Papel do QA em CI/CD
- Criar testes automáticos confiáveis e rápidos.  
- Garantir que falhas bloqueiam deploys inseguros.  
- Integrar relatórios (ex.: Allure) para visibilidade da qualidade.  
- Ajudar a manter pipelines estáveis e reprodutíveis.

---

📌 *Conceitos essenciais para QA moderno: simples, práticos e aplicáveis ao dia a dia.*
