# 🟩 QAOps

Este documento resume as ferramentas e práticas que ligam QA à automação e ao DevOps.  
Focado no essencial para um QA Automation Engineer e alinhado com a Academia.

---

## 🧩 Jenkins
- Ferramenta de automação usada para criar **pipelines de CI/CD**.  
- Executa testes automaticamente após commits ou pull requests.  
- Permite configurar jobs para UI, API e relatórios.  
- Suporta plugins para integração com Git, Docker e Allure.  
- Ajuda a garantir feedback rápido sobre a qualidade do código.

---

## 🧩 Allure Reports
- Framework de relatórios para testes automáticos.  
- Gera dashboards com passos, screenshots e anexos.  
- Facilita análise de falhas e histórico de execuções.  
- Integra com Playwright, Jenkins e GitHub Actions.  
- Torna os resultados mais claros para equipas técnicas e não técnicas.

---

## 🧩 Docker
- Plataforma para criar e executar aplicações em **containers isolados**.  
- Permite garantir que testes correm sempre no mesmo ambiente.  
- Usado em pipelines para criar ambientes limpos e reproduzíveis.  
- Evita problemas de “funciona na minha máquina”.

---

## 🧩 GitHub Actions
- Sistema de CI/CD integrado no GitHub.  
- Permite executar testes automaticamente em cada push ou PR.  
- Suporta workflows para UI, API, linting e relatórios.  
- Fácil de configurar e ideal para projetos open-source ou pessoais.
