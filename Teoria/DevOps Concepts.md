# 🟦 DevOps Concepts

Este documento resume os conceitos essenciais de DevOps aplicados ao trabalho de um QA Automation Engineer.  
Focado no que é realmente usado na prática e no que foi abordado na formação CRE.

---

## 🧩 Git & GitHub
- Sistema de controlo de versões usado para gerir código e histórico de alterações.  
- Permite criar **branches** para desenvolver funcionalidades isoladas.  
- **Commits** registam alterações pequenas e frequentes.  
- **Pull Requests** permitem revisão de código e integração controlada.  
- **Merge** junta branches; **rebase** mantém histórico limpo.  
- Resolução de **conflitos** faz parte do fluxo normal de equipa.  
- GitHub aloja repositórios, gere issues, PRs e integra com CI/CD.

---

## 🧩 Azure DevOps
- Plataforma usada para gerir projetos, código e pipelines.  
- **Boards** organizam tarefas, bugs e user stories.  
- **Repos** armazenam código com políticas de segurança e revisão.  
- **Pipelines** automatizam builds, testes e deploys.  
- Permite integrar testes automáticos em CI/CD.  
- Usado em equipas ágeis para garantir rastreabilidade e qualidade.
