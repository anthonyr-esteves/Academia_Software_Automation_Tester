Proposta de Sistema — Banking API para Treino de QA

Visão Geral

Este documento descreve a ideia inicial de um sistema simples de banco digital, cujo principal objetivo é servir como ambiente de treino para QA entry level, com foco em teste de API, teste manual, regressão e documentação de testes.

O sistema deve ser deliberadamente simples, mas completo o suficiente para permitir:

  • Teste de operações CRUD em API REST.
  • Teste de fluxos de negócio comuns (depósito, transferência).
  • Teste de frontends consumindo a mesma API (web desktop + web mobile-like).
  • Produção de artefatos de QA (plano de teste, casos de teste, relatórios etc.).

Objetivos do Sistema

  • Criar uma API REST de banco com:
    • CRUD de conta bancária.
    • Operação de depósito (receber dinheiro).
    • Operação de transferência (enviar dinheiro).
  • Expor a API com documentação interativa (Swagger).
  • Criar dois frontends simples:
    • Interface web “desktop”.
    • Interface web com layout “mobile-like”.
  • Servir como base prática para:
    • Treinar QA em teste manual de API.
    • Demonstrar testes com Postman/Swagger.
    • Exercitar conceitos de plano de teste, regressão, cobertura e bug report.

Escopo Inicial (High Level)

Backend

  • Node.js + Express.
  • Armazenamento em memória (sem BD real).
  • Endpoints básicos para contas e transações.

Frontend Web

  • CRUD de contas e operações de depósito/transferência.
  • Visualização em tabela das contas.

Frontend Mobile-like

  • Visualização em layout de “tela de celular”.
  • Card de saldo total.
  • Formulários simplificados para as mesmas operações da API.

Não Escopo (Neste Momento)

  • Autenticação/autorização.
  • Segurança avançada (tokens, criptografia).
  • Persistência em banco de dados.
  • Performance e escalabilidade.
  • Testes automatizados de código (unitários ou E2E) — foco é teste manual.
