Backlog de Tarefas — Banking API Training System

Este documento traduz a especificação funcional completa em tarefas de desenvolvimento. As 10 primeiras tarefas estão detalhadas (prontas para desenvolvimento) e as demais aparecem apenas com título, compondo o backlog para refinamento e estimativa futura.

Top 10 tarefas detalhadas (MVP)

  • TASK-001 Módulo: Backend Backend Alta prioridade
    Configurar projeto Node.js + Express e servidor base

    Descrição: Criar a estrutura inicial do projeto backend, configurar o Express e expor um endpoint raiz para verificação de saúde do servidor.

    Critérios de aceite:

    • Projeto Node.js criado com package.json e dependências principais (express, cors).
    • Servidor iniciado com npm run dev ouvindo na porta 3000.
    • Endpoint GET / responde com JSON simples ({ "message": "Banking API rodando" }).
  • TASK-002 Módulo: Backend Backend Alta prioridade
    Implementar modelo em memória para armazenamento de contas

    Descrição: Criar estrutura simples em memória para armazenar as contas bancárias, contendo os campos definidos na especificação (id, name, email, cpf, balance, createdAt).

    Critérios de aceite:

    • Arquivo dedicado (ex.: db.js) expondo um array de contas.
    • Capacidade de inserir, ler e remover objetos de conta nesse array.
    • Nenhuma dependência de banco de dados externo (tudo em memória).
  • TASK-003 Módulo: Backend Backend Alta prioridade
    Implementar endpoint POST /accounts (criar conta)

    Descrição: Implementar o endpoint de criação de conta conforme a especificação, incluindo validação de campos obrigatórios, validação de saldo inicial e unicidade de CPF.

    Critérios de aceite:

    • POST /accounts aceita name, email, cpf, initialBalance.
    • Retorna 201 com objeto da conta criada e campos completos (id, balance, createdAt etc.).
    • Retorna 400 se faltar campo obrigatório ou se initialBalance < 0.
    • Retorna 400 se o cpf já existir em alguma conta em memória.
  • TASK-004 Módulo: Backend Backend Alta prioridade
    Implementar endpoints GET /accounts e GET /accounts/{id}

    Descrição: Implementar os endpoints de listagem de todas as contas e busca por ID, incluindo tratamento de conta não encontrada.

    Critérios de aceite:

    • GET /accounts retorna array de contas (pode ser vazio).
    • GET /accounts/{id} retorna 200 com conta correspondente quando o ID existe.
    • GET /accounts/{id} retorna 404 com { "error": "Conta não encontrada" } quando o ID não existe.
  • TASK-005 Módulo: Backend Backend Média prioridade
    Implementar endpoint PUT /accounts/{id} (atualizar conta)

    Descrição: Implementar o endpoint de atualização de conta, permitindo alterar name e email, sem modificar cpf e balance.

    Critérios de aceite:

    • PUT /accounts/{id} aceita campos opcionais name e email.
    • Retorna 200 com conta atualizada quando o ID existe.
    • Retorna 404 com mensagem de erro quando a conta não existe.
    • Valores de cpf e balance permanecem inalterados após a atualização.
  • TASK-006 Módulo: Backend Backend Média prioridade
    Implementar endpoint DELETE /accounts/{id} (remover conta)

    Descrição: Implementar a remoção de conta em memória, retornando mensagens adequadas para sucesso e erro.

    Critérios de aceite:

    • DELETE /accounts/{id} remove a conta do array quando o ID existe.
    • Retorna 200 com { "message": "Conta removida com sucesso" } em caso de sucesso.
    • Retorna 404 com { "error": "Conta não encontrada" } se a conta não existir.
  • TASK-007 Módulo: Backend Backend Alta prioridade
    Implementar endpoint POST /accounts/{id}/deposit (depósito)

    Descrição: Implementar a funcionalidade de depósito, incluindo validação de valor e tratamento de conta inexistente.

    Critérios de aceite:

    • POST /accounts/{id}/deposit aceita { "amount": number }.
    • Retorna 200 com { "message": "Depósito realizado", "balance": <novoSaldo> } quando amount > 0 e conta existe.
    • Retorna 400 com mensagem de erro quando amount <= 0.
    • Retorna 404 com erro de conta não encontrada quando o ID não existe.
  • TASK-008 Módulo: Backend Backend Alta prioridade
    Implementar endpoint POST /accounts/{id}/transfer (transferência)

    Descrição: Implementar a transferência entre contas, com validações de saldo, conta destino, valor positivo e proibição de transferir para a mesma conta.

    Critérios de aceite:

    • POST /accounts/{id}/transfer aceita { "targetId": string, "amount": number }.
    • Valida existência das contas origem e destino.
    • Retorna 200 com { "message": "Transferência realizada com sucesso", "sourceBalance": ..., "targetBalance": ... } em caso de sucesso.
    • Retorna 400 para:
      • Saldo insuficiente.
      • amount <= 0.
      • Origem e destino iguais.
    • Retorna 404 se origem ou destino forem inexistentes.
  • TASK-009 Módulo: Backend / Documentação Backend Doc Média prioridade
    Configurar Swagger UI e documentar todos os endpoints

    Descrição: Integrar Swagger UI ao projeto e documentar todos os endpoints da Banking API, incluindo modelos de request/response e exemplos.

    Critérios de aceite:

    • Swagger disponível em http://localhost:3000/api-docs.
    • Todos os endpoints (/accounts, /accounts/{id}, /deposit, /transfer) documentados com:
      • Resumo e descrição.
      • Parâmetros de path e body.
      • Códigos de resposta esperados (200, 201, 400, 404).
    • É possível executar chamadas pela interface do Swagger.
  • TASK-010 Módulo: Frontend Web Frontend Alta prioridade
    Implementar página frontend-web com cards básicos e tabela de contas

    Descrição: Criar o frontend desktop em frontend-web/index.html, com cards para operações principais e tabela para listar contas, consumindo diretamente a API.

    Critérios de aceite:

    • Página exibe cards para:
      • Criar conta.
      • Atualizar conta.
      • Depósito.
      • Transferência.
      • Buscar / Deletar conta.
    • Área “Resposta da API” mostra o JSON retornado, destacando erros.
    • Seção “Todas as Contas” mostra tabela com ID abreviado, Nome, Email, CPF, Saldo.
    • Botão “Atualizar Lista” consome GET /accounts e atualiza a tabela.
